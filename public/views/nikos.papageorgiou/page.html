About Me

I love clean and orthogonal solutions. During my career I discovered that I have a rare talent: I am good at stopping, reverting, or averting in the first place the slow decline of codebases and systems towards entropy.

If we work together, you should expect to hear me talk a lot about the importance of tests, simplicity, and standards.

I somehow managed to collect 2 degrees in Business and 1 in IT. I like to think that my knowledge of Business and IT helps me create valuable solutions.

A problem-solver at heart, I dont only use code to solve problems. I put great focus on understanding the domain of the business I work in, and after (or while) I do that, I am not shy of challenging current solutions.

In my spare time I enjoy experimenting with new technology, reading history and waiting for the next Wes Anderson film.

Pet Projects

  • Crypto trading bot

    Description: How could 2017 be complete without a crypto related project? Bittrex bot is an automatic trading tool, that executes orders against Bittrex, one of the most used crypto-exchanges.

    Nerdy info: The bot implements a moving average crossover strategy. It can be configured to operate on various periods (1h, 4h, 1d, etch) depending on how often the cron job will run it, but some backtesting has shown that 1d produces the best results.

    It has produced satisfying results so far, and you are free to use it at your own risk.

    Technologies used: Php

  • Chat

    Description: Skaip is a chat program that you can host, and others can download and chat with you.

    Nerdy info: Both the website and chat server are written in standard Java, without any external library support. The chat server opens a socket for each client and handles each one of them as a thread. Client-server communication builds on top of the HTTP protocol. This was a great learning experience on the internals of HTTP and sockets.

    Technologies used: HTTP, Java

  • SeaCargo

    Description: SeaCargo is an imaginary transport company. You can view (or delete) info about its operations, so free to play (responsibly!) with it.

    Nerdy info: The website is hosted by a written-from-scratch Node Server. Objects are saved in a noSQL DB (mongo). Html is build by server-side templating (Jade), and lazy-loaded with ajax calls. This was an exercise on Node servers, noSQL, and server-side-templating.

    Technologies used: Node, noSQL, Jade, Heroku

  • GlobeJob

    Description: Globejob is project that gathers all the english-speaking job ads in Denmark. A useful tool for those who do not speak Danish.

    Nerdy info: My first big pet project. The jobs are taken by web crawlers written in Java, and persisted in an AWS database. The web server is a Node server, while the client runs on Angular. This is a really distributed project, whose components run on 3 different cloud platforms.

    Technologies used: Node, Angular, Javascript, ajax, JSON, Java, JPA, mySQL

  • Budget

    Description: Budget is project that lets you examine and present bank account transactions.

    Nerdy info: My first Ruby project. I wanted to learn Ruby, and experiment with the idea of clients of different nature hooking up to the same server. The project has 2 clients, a webpage and a console app. They collect user input any way their nature allows, format it to the standard that the Backend expects, and present the output in the way that makes sense to their nature.

    Technologies used: Angular, ajax, Ruby

  • FastRent

    Description: Who doesn't love crawlers? FastRent is currently under development, and fetches all the latest housing ads in Denmark.

    Nerdy info: My first large Ruby project. I wanted to avoid the Rails overhead, so I decided to build the app from scratch: Puma for the server, sinatra for endpoints, pg(postgres) and active record for database communication and ORM. Apart from that, the system includes processes for crawling the web and populating the database, for cleaning up expired ads, and for sending emails at scheduled intervals. In the close future, it will also have a whole bunch of other integrations with external services, such as SMS and online payment.

    Technologies used: Angular, Ruby, Postgres, Heroku
